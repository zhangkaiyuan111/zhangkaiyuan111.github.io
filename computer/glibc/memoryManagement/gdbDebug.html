1.4 gdb调试

GDB 调试

GDB 环境设置

  1. 进入gdb
    x@ubuntu:~/work/libs/libraries$ gdb ./output/build/app/glibc/tmem
  2. 设置调试源代码路径
    (gdb) directory package/glibc-2.37/malloc/
  3. 显示调试源代码路径
    (gdb) show directories
    Source directories searched: /home/x/work/libs/libraries/package/glibc-2.37/malloc:$cdir:$cwd
  4. 设置断点
    (gdb) b main
  5. run 运行
    (gdb) r
  6. Step Into 单步跟踪进入，n(Step Over，单步跟踪)
    (gdb) s

使用 GDB 查看程序的栈空间

  • bt(backtrace) 命令：查看函数调用的顺序，即当前函数向上的所有函数调用栈信息。
  • info frame 命令：与 bt 命令不同，i frame 命令查看当前函数调用的栈帧信息。而不是所有函数调用堆栈信息。
#include <stdlib.h>
#include <stdio.h>
static void t() {
    char* p2 = malloc(32);
    snprintf(p2, 32, "hello, I am here 2");
}

int main(int argc, char **argv) {
    char* p1 = malloc(32);
    snprintf(p1, 32, "hello, I am here 1");
    printf("111111111111\n");
    t();

    return 0;
}
